Domanda 1: Web Service vs Software-Oriented Architecture (SOA)

  • 1. Web service sono lo standard principale per realizzare SOA. VERO
  • 2. SOA sono componenti loosely coupled. VERO
  • 3. Web service è uno stile architetturale per implementare applicazioni software che usano servizi disponibili in rete. FALSO
Motivazione: La 3 è falsa perché "Web Service" non è uno stile architetturale, bensì una tecnologia/implementazione concreta (tramite SOAP o REST). Lo stile architetturale descritto (basato su servizi esposti in rete) è proprio SOA (Service-Oriented Architecture). L'affermazione 2 è vera: i servizi in SOA sono disaccoppiati (loosely coupled).

Domanda 2: HTML DOM

  • 1. È uno standard che definisce la struttura di una pagina HTML. FALSO
  • 2. È usato da JavaScript per modificare gli elementi di una pagina HTML. VERO
  • 3. Rappresenta una pagina HTML come un grafo ordinato aciclico. FALSO
Motivazione: La 1 è tecnicamente imprecisa (è l'HTML stesso a definire la struttura; il DOM è l'interfaccia/modello a oggetti per interagirci). La 3 è falsa: il DOM modella la pagina non come un Grafo Diretto Aciclico (DAG) generico, ma specificamente come un Albero (Tree) gerarchico (padre, figli, fratelli).

Domanda 3: XML Remote Procedure Call (XML-RPC)

  • 1. Fornisce un protocollo di richiesta-risposta. VERO
  • 2. Supporta tipi complessi come Array e Struct. VERO
  • 3. È alla base dei Web Service REST. FALSO
Motivazione: La 3 è falsa in quanto XML-RPC (basato su chiamate remote a metodi) è il precursore e la base tecnologica del protocollo SOAP, non di REST (che è uno stile architetturale radicalmente diverso orientato alle risorse e non ai metodi).

Domanda 4: Si considerino i seguenti codici XML e JavaScript

XML / JS
<menu>
  <menu>
    <primi>
      <primo>lasagna vegana</primo>
      <primo>zuppa di fagioli</primo>
      <primo>risotto agli asparagi</primo>
    </primi>
    <secondi>
      <secondo>verdure miste</secondo>
      <secondo>parmigiana</secondo>
      <secondo>pomodori ripieni</secondo>
    </secondi>
    <dessert>frutta</dessert>
  </menu>
  <menu>
    <!-- Secondo menu interno (omesso per brevità: lasagna, spaghetti allo scoglio, ecc...) -->
    <dessert>tiramisu</dessert>
  </menu>
</menu>

<script>
  // (Richiesta XMLHTTP omessa per brevità)
  xmlDocMenu = xmlDoc.getElementsByTagName("menu");
  
  document.write("<div>");
  // Stampa 1
  document.write(xmlDocMenu[1].getElementsByTagName("primo")[2].childNodes[0].nodeValue + "<br>");
  // Stampa 2
  document.write(xmlDocMenu[1].getElementsByTagName("secondo")[1].childNodes[0].nodeValue + "<br>");
  // Stampa 3
  document.write(xmlDoc.getElementsByTagName("menu")[0].getElementsByTagName("dessert")[0].childNodes[0].nodeValue + "<br>");
  document.write("</div>");
</script>
  • 1. La funzione JavaScript stampa "risotto agli asparagi, parmigiana, frutta". VERO
  • 2. La funzione JavaScript stampa "spaghetti allo scoglio, polpette, undefined". FALSO
  • 3. La funzione JavaScript stampa "risotto alla salsiccia, bistecca alla milanese, frutta". FALSO
Motivazione (Trace del DOM XML): Il tag root è <menu>, che a sua volta contiene due tag <menu> figli. getElementsByTagName("menu") restituisce un array: l'indice [0] è la root, l'indice [1] è il primo menu figlio (vegano).
- Stampa 1: Cerca il "primo" all'indice [2] nel menu [1] -> "risotto agli asparagi".
- Stampa 2: Cerca il "secondo" all'indice [1] nel menu [1] -> "parmigiana".
- Stampa 3: Cerca "dessert" all'indice [0] partendo dalla root (menu[0]). Il metodo esegue una ricerca in profondità e trova il primissimo dessert del documento -> "frutta".

Domanda 5: Si consideri il seguente codice CSS3

CSS
<style>
div {
  width: 100px;
  height: 100px;
  background-color: red;
  position: relative;
  animation-name: example;
  animation-duration: 4s;
  animation-iteration-count: infinite;
}
@keyframes example {
  0% {background-color:red; left: 0px; top: 0px;}
  50% {background-color:blue; left: 200px; top:200px;}
  100% {background-color: red; left: 0px; top: 0px;}
}
</style>
  • 1. Qualsiasi elemento div viene traslato avanti e indietro unicamente lungo l'asse delle x. FALSO
  • 2. Solo il primo elemento div definito all'interno della pagina HTML cambia colore. FALSO
  • 3. L'animazione viene eseguita una volta sola per 4 secondi. FALSO
  • 4. L'elemento div non cambia dimensione. VERO
Motivazione: La 1 è falsa perché variano le coordinate di left (asse X) ma anche di top (asse Y), quindi il movimento è diagonale. La 2 è falsa: il selettore "div" si applica a tutti i div della pagina. La 3 è falsa per via della direttiva animation-iteration-count: infinite che ripete il loop all'infinito. La 4 è vera: l'animazione cambia posizione e colore, ma larghezza e altezza restano fisse a 100px.

Domanda a Risposta Aperta: Fogli di stile CSS e Box Model

Dopo aver descritto le caratteristiche principali e a cosa servono i fogli di stile CSS, mostrando la sintassi e un esempio di regola CSS, si discuta in dettaglio il CSS box model.

Soluzione:
I CSS (Cascading Style Sheets) sono un linguaggio dedicato alla presentazione visiva dei documenti HTML. Il loro scopo principale è separare la formattazione (layout, colori, font) dalla struttura logica della pagina (HTML). Questo rende il codice più pulito, mantenibile e permette di applicare lo stesso stile a migliaia di pagine usando un solo file esterno (.css).

Sintassi ed Esempio: Una regola CSS è composta da un selettore (che indica l'elemento da modificare) e un blocco di dichiarazioni racchiuse tra parentesi graffe.
h1 { color: red; font-size: 24px; } (Imposta tutti i tag <h1> con testo rosso e grandezza 24px).

CSS Box Model: È il modello di base utilizzato dai browser per disporre gli elementi in una pagina. Ogni elemento HTML è considerato come una "scatola" composta da 4 livelli concentrici:
1. Content: L'area centrale effettiva in cui compare il testo o l'immagine (dimensione gestita da width e height).
2. Padding: Un margine interno e trasparente che distanzia il Content dal bordo.
3. Border: La linea visibile che chiude il Padding e il Content (si possono definirne spessore e colore).
4. Margin: L'area esterna, anch'essa trasparente, che separa il "Box" in questione dagli altri elementi HTML circostanti. La dimensione totale visibile di un elemento sarà sempre data da: Width + Padding(L+R) + Border(L+R).
