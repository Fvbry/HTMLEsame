Domanda 1: Si consideri il seguente codice HTML/JavaScript

HTML / JS
<html>
<body>
  <h1 id="pizzeria">Pizzeria AWC</h1>
  <h3 id="ing">Ingredienti</h3>
  <p id="pizza"> Pizza Margherita <span id="q1">Baby</span> </p>
  <p id="ing1"> Mozzarella <span id="q1">300g</span></p>
  <p id="ing2"> Salsa di pomodoro <span id="q2">200g</span></p>
  <p id="ing3"> Pasta per pizza <span id="q3">400g</span></p>
  <h3 id="preparazione">Preparazione</h3>
  <div id="ricetta"></div>

  <script>
    myText = document.getElementById("pizza").childNodes[1].childNodes[0].nodeValue;
    
    var para = document.createElement("p");
    var node = document.createTextNode("Preparazione per la pizza " + myText);
    para.appendChild(node);
    
    var element = document.getElementById("preparazione");
    element.innerHTML = para.childNodes[0].nodeValue;
    
    // NOTA: Si assume getElementsByTagName("p") altrimenti il codice successivo fallirebbe
    myText2 = document.getElementsByTagName("p"); 
    
    var para = document.createElement("p");
    var node = document.createTextNode("Stendi la " + myText2[1].childNodes[0].nodeValue);
    para.appendChild(node);
    
    var node = document.createTextNode("Aggiungi " + myText2[2].childNodes[1].childNodes[0].nodeValue + " di " + myText2[2].childNodes[0].nodeValue);
    para.appendChild(node);
    
    var node = document.createTextNode("Ottieni una pizza " + myText2[0].childNodes[0].nodeValue);
    para.appendChild(node);
    
    var element = document.getElementById("ricetta");
    element.appendChild(para);
  </script>
</body>
</html>

Quali delle seguenti immagini rappresenta la visualizzazione esatta? (a, b o c?)

  • Risposta corretta: 1. a (L'immagine che riporta: "Stendi la Mozzarella Aggiungi 200g di Salsa di pomodoro Ottieni una pizza Pizza Margherita") CORRETTA
Motivazione (Analisi del DOM):
1. myText estrae la stringa "Baby" dal tag span interno al paragrafo con id "pizza". La scritta "Preparazione" viene quindi sovrascritta con "Preparazione per la pizza Baby".
2. Viene creato un nuovo paragrafo para per la ricetta.
3. myText2[1].childNodes[0].nodeValue accede al testo del secondo paragrafo (" Mozzarella "). -> "Stendi la Mozzarella".
4. myText2[2] accede a "Salsa di pomodoro" (childNodes[0]) e al suo span "200g" (childNodes[1]). -> "Aggiungi 200g di Salsa di pomodoro".
5. myText2[0].childNodes[0].nodeValue accede al primo testo di "pizza": " Pizza Margherita ". -> "Ottieni una pizza Pizza Margherita".
L'immagine "a" è l'unica che concatena correttamente i testi.

Domanda 2: Si consideri il seguente brano di codice XML

XML / SOAP
<soap:Envelope xmlns:soap="http://www.w3.org/2003/05/soap-envelope/" soap:encodingStyle="...">
  <soap:Body>
    <m:GetPrice xmlns:m="https://www.awc.com/prices">
      <m:Item>One Plus 5</m:Item>
    </m:GetPrice>
  </soap:Body>
</soap:Envelope>
  • 1. È parte di una specifica di interfaccia WSDL. FALSO
  • 2. È parte di una richiesta SOAP. VERO
  • 3. Viene usato da un web service che definisce interfacce standard HTTP. FALSO
Motivazione: Il codice mostra chiaramente la struttura "Envelope" e "Body", tipica del payload di un messaggio SOAP. La 1 è falsa perché WSDL descrive l'interfaccia (il contratto), non il payload della singola chiamata. La 3 è falsa perché i servizi SOAP non usano interfacce standard HTTP (come REST che usa GET, PUT, POST), ma definiscono operazioni (metodi) custom (es. GetPrice) nascoste all'interno del Body XML.

Domanda 3: In un RESTful Web Service

  • 1. Le risorse vengono identificate attraverso file XML. FALSO
  • 2. Le risorse vengono accedute attraverso un'interfaccia uniforme basata su HTTP operation. VERO
  • 3. Le operazioni GET, HEAD, PUT, DELETE sono idempotenti. VERO
  • 4. Le operazioni PUT e POST hanno lo stesso comportamento. FALSO
Motivazione: La 1 è falsa perché le risorse sono identificate tramite gli URI (Uniform Resource Identifier), non file XML. La 3 è vera: GET, HEAD, PUT e DELETE garantiscono l'idempotenza (eseguirle più volte produce lo stesso risultato di eseguirle una volta sola). La 4 è falsa perché POST non è idempotente (crea nuove risorse).

Domanda 4: XML vs JSON

  • 1. La sintassi JSON è un sottoinsieme della sintassi JavaScript. VERO
  • 2. XML è usato per descrivere dati strutturati che includono array. FALSO
  • 3. JSON è più verbose di XML. FALSO
  • 4. JSON è usato come base dei protocolli per i servizi web basati su SOAP. FALSO
Motivazione: La 1 è vera (JSON sta per JavaScript Object Notation e nasce proprio dalla notazione degli oggetti letterali in JS). La 2 è falsa perché XML nativamente non ha il tipo di dato "Array". La 3 è falsa perché JSON è molto meno verboso (più compatto). La 4 è falsa perché SOAP si basa esclusivamente su XML.

Domanda 5: I fogli di stile CSS

  • 1. Definiscono lo stile di visualizzazione/formattazione. VERO
  • 2. Utilizzano il selettore id per identificare lo stile di un singolo elemento all'interno della pagina HTML. VERO
  • 3. Definiscono delle regole con la sintassi "selettore {proprietà: valore, ...}". VERO (concettualmente)
Motivazione: L'opzione 3 viene data genericamente come Vera per indicare che il blocco si apre con le parentesi graffe {}. Attenzione tecnica: il separatore reale tra le proprietà CSS non è la virgola, ma il punto e virgola ;. Tuttavia, rispetto a esami precedenti (dove venivano proposte parentesi quadre errate), qui la sintassi base selettore { ... } è considerata corretta per definizione.

Domanda 6: HTML vs XHTML

  • 1. XHTML ha regole sintattiche più stringenti di HTML. VERO
  • 2. XHTML e HTML sono molto diversi tra loro. FALSO
  • 3. XHTML combina HTML e JSON. FALSO
Motivazione: XHTML (Extensible HyperText Markup Language) è sostanzialmente HTML formattato seguendo le rigidissime regole sintattiche di XML (es. tag scritti in minuscolo, valori degli attributi sempre tra virgolette, tag di chiusura sempre obbligatori). Non sono "molto diversi" funzionalmente (i tag sono gli stessi), ma solo sintatticamente. JSON non c'entra nulla con XHTML.

Domanda a Risposta Aperta: CSS3 e Media Rule

Dopo aver descritto le caratteristiche principali e a cosa serve CSS3, si discutano in dettaglio le media rule.

Soluzione:
CSS3 è l'ultima evoluzione del linguaggio CSS, utilizzato per definire lo stile e la formattazione visiva dei documenti HTML. CSS3 è stato suddiviso in moduli indipendenti e ha introdotto novità straordinarie come gli sfondi multipli, i bordi arrotondati, l'opacità, i Web Fonts (@font-face) e le animazioni/transizioni senza bisogno di JavaScript.

Le Media Rule (Media Queries) sono la base fondamentale per il Responsive Web Design (RWD). Introdotte con CSS3, permettono di applicare blocchi di codice CSS diversi a seconda delle caratteristiche del dispositivo su cui la pagina viene visualizzata (come la larghezza dello schermo, l'altezza, la risoluzione o l'orientamento).
La sintassi si basa sulla direttiva @media seguita da una condizione. Se la condizione è vera, il blocco di stili al suo interno viene eseguito sovrascrivendo gli stili precedenti.

Esempio:
body { background-color: blue; }
@media screen and (max-width: 600px) {
  body { background-color: red; }
}
In questo esempio, il colore di sfondo sarà blu su desktop, ma cambierà automaticamente in rosso sui dispositivi mobili (schermi più piccoli di 600px).
