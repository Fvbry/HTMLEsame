Domanda 1: Si consideri il seguente codice JavaScript

JAVASCRIPT
function exec_f(f, x) {
  return f(x);
}
alert(exec_f(new Function ("x", "return x+x/2"), 10));
  • 1. La funzione alert mostra un errore. FALSO
  • 2. La funzione alert mostra 15. VERO
  • 3. La funzione alert mostra 10. FALSO
Motivazione: La sintassi new Function("x", "return x+x/2") genera dinamicamente una funzione anonima che prende un parametro x e ritorna x + x/2. Quando viene invocata tramite exec_f passando 10 come argomento, l'operazione eseguita è 10 + 10/2 = 10 + 5 = 15. Quindi l'alert stamperà 15.

Domanda 2: La valutazione a cascata delle regole CSS prevede che

  • 1. Il primo ordinamento sia fatto in base a peso e origine. VERO
  • 2. Selettori più specifici sovrascrivano selettori più generali. VERO
  • 3. Per dichiarazioni "importanti", gli stylesheet dell'autore della pagina sovrascrivano quelli dell'utente. FALSO
Motivazione: L'opzione 3 è falsa perché, secondo le specifiche del W3C CSS2.1+, per questioni di accessibilità, le dichiarazioni segnate come !important dall'utente sovrascrivono sempre le dichiarazioni !important dell'autore del sito.

Domanda 3: HTML DOM

  • 1. Rappresenta una pagina web come un grafo. FALSO
  • 2. Viene usato per il parsing di documento XML. FALSO
  • 3. Definisce gli elementi HTML come oggetti. VERO
  • 4. È un modo diverso di riferirsi a XHTML. FALSO
Motivazione: La 1 è falsa perché il DOM rappresenta la pagina come un Albero (Tree), non come un generico grafo. La 2 è falsa perché per l'XML si utilizza il DOM XML, mentre qui si parla specificamente di HTML DOM. La 4 non c'entra niente (XHTML è un linguaggio di markup XML-based, il DOM è un'API).

Domanda 4: I web service basati su SOAP

  • 1. Implementano un paradigma client-server. VERO
  • 2. Sono indipendenti dalla piattaforma. VERO
  • 3. Sono componenti "strongly coupled". FALSO
Motivazione: L'affermazione 3 è falsa. L'architettura SOA (Software-Oriented Architecture) su cui si fonda tipicamente SOAP è stata pensata proprio per integrare sistemi distribuiti eterogenei promuovendo il disaccoppiamento (loosely coupled) tra fornitore del servizio e consumatore.

Domanda 5: Si consideri il seguente codice CSS3

CSS
<style>
div {
  position: relative;
  background-color: blue;
  width: 150px;
  height: 300px;
  animation-name: exam-trs;
  animation-duration: 10s;
}
@keyframes exam-trs {
  0% {background-color: blue; left: 0px; top:0px;}
  25% {background-color:red; left: 0px; top:200px; height: 150px;}
  50% {background-color:green; left: 200px; top:200px;}
  75% {background-color:red; left: 200px; top:0px;}
  100% {background-color: blue; left: 0px; top: 0px;}
}
</style>
  • 1. Qualsiasi elemento div viene traslato in senso antiorario. VERO
  • 2. Solo il primo elemento div definito all'interno della pagina HTML cambia colore. FALSO
  • 3. L'animazione viene eseguita una volta sola per 10 secondi. VERO
  • 4. L'elemento div non cambia dimensione. FALSO
Motivazione:
1. VERO: Il div parte da (0,0), va in basso a (0,200), poi a destra a (200,200), poi in alto a (200,0) e torna a (0,0). Questo definisce un movimento perimetrale quadrato in senso antiorario.
2. FALSO: Il selettore div si applica a tutti i div della pagina.
3. VERO: Non essendoci animation-iteration-count specificato, il valore di default è 1 (viene eseguita una volta sola per 10 secondi totali).
4. FALSO: Al 25% dell'animazione la proprietà height cambia diventando 150px.

Domanda a Risposta Aperta 1: Novità di HTML5

Si discutano nel dettaglio le principali funzionalità introdotte con l'avvento di HTML5.

Soluzione:
HTML5 ha rivoluzionato il Web introducendo potenti funzionalità che ne hanno aumentato sia la semantica che le capacità applicative lato client:
- Tag Semantici: Introduzione di tag come <header>, <footer>, <article>, e <nav> che rendono il documento più leggibile per motori di ricerca e screen reader, a scapito dei classici e anonimi <div>.
- Supporto Multimediale Nativo: I tag <audio> e <video> permettono di riprodurre flussi multimediali senza richiedere plugin esterni di terze parti come Adobe Flash.
- Nuove API e Grafica: L'introduzione del tag <canvas> fornisce una superficie su cui JavaScript può disegnare grafica 2D e 3D in tempo reale. Sono state aggiunte anche API per la Geolocalizzazione, il Drag & Drop nativo e i Web Workers per il multithreading.
- Web Storage: Un miglioramento radicale per la memorizzazione dei dati sul browser dell'utente (Session Storage e Local Storage) al fine di sostituire la gestione macchinosa e limitata dei classici cookie.

Domanda a Risposta Aperta 2: REST - POST vs PUT

Nell'ambito delle architetture REST, si discutano le principali differenze tra le operazioni POST e PUT. Si presenti un esempio per ognuna delle operazioni.

Soluzione:
Entrambi i metodi servono per inviare dati al server e manipolare risorse, ma presentano una differenza sostanziale legata all'idempotenza (la proprietà per cui eseguire l'operazione una o N volte produce lo stesso stato finale).

- POST: Non è idempotente. Viene utilizzato per creare una nuova risorsa subordinata a un'altra. In questo caso è il server a determinare l'URI della nuova risorsa (assegnandole un ID). Ripetere la stessa POST più volte creerà risorse duplicate.
Esempio POST: POST /utenti (invia i dati di un nuovo utente; il server lo creerà all'URI /utenti/101).

- PUT: È idempotente. Viene utilizzato per aggiornare o sostituire interamente una risorsa già esistente fornendo una nuova rappresentazione. Può essere usato anche per creare una risorsa se il client è a conoscenza in anticipo dell'URI esatto (cioè decide lui l'ID). Ripetere la stessa PUT N volte sovrascrive semplicemente l'elemento con lo stesso stato.
Esempio PUT: PUT /utenti/101 (invia il profilo aggiornato dell'utente 101, che andrà a sovrascrivere i dati preesistenti in quell'URI).
