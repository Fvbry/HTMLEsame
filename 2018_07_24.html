Domanda 1: I fogli di stile CSS

  • 1. Richiedono versioni HTML precedenti a HTML5. FALSO
  • 2. Definiscono lo stile di visualizzazione/formattazione. VERO
  • 3. Definiscono delle regole con la sintassi "selettore [proprietà: valore, ...]". FALSO
  • 4. Utilizzano il selettore id per identificare lo stile di un singolo elemento all'interno della pagina HTML. VERO
Motivazione: La 1 è falsa perché CSS è pienamente supportato ed essenziale in HTML5. La 3 è tecnicamente falsa a causa della sintassi riportata: in CSS le regole si definiscono racchiudendo le proprietà tra parentesi graffe { } e separate da punto e virgola ;, non tra parentesi quadre con la virgola.

Domanda 2: Si consideri il seguente codice CSS

CSS
div {
                    width: 200px;
                    height: 200px;
                    background: blue;
                    transition: width 4s, height 2s, transform 2s;
                    }
                    div:hover {
                    width: 400px;
                    height: 400px;
                    transform: rotate(90deg);
}
  • 1. Al passare del mouse sopra l'elemento div, ingrandisce l'elemento div. VERO
  • 2. Definisce un'animazione basata su HTML5. FALSO
  • 3. Al passare del mouse sopra l'elemento div, inizia un'animazione della durata di 8 secondi. FALSO
  • 4. Al passare del mouse sopra l'elemento div, sposta l'elemento div verso destra. FALSO
Motivazione: La 1 è vera (larghezza e altezza passano da 200px a 400px). La 2 è falsa perché si tratta di CSS3, non HTML5. La 3 è falsa: le transizioni partono in contemporanea, quindi la durata totale dell'effetto corrisponde a quella della transizione più lunga, ovvero 4 secondi (non la somma di 4+2+2). La 4 è falsa perché il div viene ruotato sul proprio asse (rotate 90deg), non spostato (nessun translateX o left).

Domanda 3: Il semantic markup in HTML5

  • 1. Rende una pagina web comprensibile sia all'uomo che alla macchina. VERO
  • 2. Migliora l'usabilità per persone con disabilità. VERO
  • 3. Associa un significato al contenuto. VERO
Motivazione: L'uso di tag semantici (es. <article>, <nav>) non solo rende il codice più chiaro per i programmatori e i motori di ricerca, ma è fondamentale per le tecnologie assistive (come gli screen reader) che aiutano gli utenti ipovedenti a navigare correttamente la struttura della pagina.

Domanda 4: Si consideri il seguente codice JavaScript

HTML / JS
<FORM NAME="modulo" onSubmit="print(); return false;">
                    <SELECT NAME="menu" id="menuid">
                        <OPTION ID="op1" VALUE="stufato">Stufato di manzo</OPTION>
                        <OPTION ID="op2" VALUE="tagliata">Tagliata</OPTION>
                    </SELECT>
                    <INPUT TYPE="Submit" VALUE="OK">
                    </FORM>

                    <SCRIPT language="JavaScript">
                        var city = prompt("Sei carnivoro o vegetariano?", "Carnivoro");
                        if(city == "Vegetariano") {
                        window.document.modulo.menu.options[0].text="Minestra";
                        window.document.modulo.menu.options[1].text="Patate al forno";
                        
                        var select = document.getElementById('menuid');
                        var opt = document.createElement('option');
                        opt.id = "op3";
                        opt.value = "pasta";
                        opt.innerHTML = "Pasta aglio e olio";
                        select.appendChild(opt);
                        }
                    </SCRIPT>
  • 1. Minestra in posizione 0 ove presente. VERO
  • 2. Stufato di manzo e Tagliata sempre presenti tra le opzioni. FALSO
  • 3. Pasta aglio e olio sempre dopo Stufato di manzo. FALSO
  • 4. Pasta aglio e olio e Stufato di manzo mai insieme. FALSO
Motivazione (max 6 righe):
1. VERO: se l'utente inserisce "Vegetariano", "Minestra" sovrascrive l'elemento all'indice 0.
2. FALSO: se l'utente è "Vegetariano", Stufato e Tagliata vengono sovrascritti.
3. FALSO: se l'utente è "Vegetariano", "Stufato di manzo" scompare, quindi la pasta non può trovarsi "dopo" di esso.
4. FALSO: se l'utente risponde qualsiasi cosa diversa da "Vegetariano" (es. "Carnivoro"), lo script non entra nell'if. Stufato rimane e "Pasta aglio e olio" viene aggiunto alla fine, coesistendo entrambi.

Domanda a Risposta Aperta 1: HTML DOM e JavaScript

Nell'ambito del linguaggio di scripting JavaScript, si descriva il ruolo dell'HTML DOM per la modifica degli elementi di una pagina web e si proponga un esempio del suo utilizzo.

Soluzione:
L'HTML DOM (Document Object Model) è un'API e un modello ad albero che rappresenta la struttura del documento HTML in memoria. Il suo ruolo fondamentale è permettere ai linguaggi di scripting, in particolare a JavaScript, di accedere a tutti gli elementi della pagina in modo dinamico. Tramite il DOM, JavaScript può leggere o modificare i contenuti (testo), cambiare lo stile (CSS), e aggiungere, spostare o rimuovere tag (nodi) in risposta alle azioni dell'utente, senza ricaricare la pagina.

Esempio di utilizzo:
<p id="demo">Vecchio Testo</p>
<button onclick="document.getElementById('demo').innerHTML = 'Nuovo Testo!'">Cambia</button>

Domanda a Risposta Aperta 2: SOAP Web Service e WSDL

Dopo aver introdotto le caratteristiche principali dei SOAP Web Service, si descriva il ruolo del protocollo WSDL.

Soluzione:
I SOAP Web Service si basano su un protocollo (Simple Object Access Protocol) di comunicazione molto rigoroso e formale per lo scambio di dati strutturati (esclusivamente in formato XML) tra macchine su una rete. Sono "platform independent", ma fortemente strutturati in Envelope, Header e Body.

Il WSDL (Web Services Description Language) è un documento in formato XML che svolge un ruolo fondamentale: funge da "contratto" pubblico del servizio. Il suo scopo è descrivere cosa fa il Web Service e come comunicare con esso. Definisce in dettaglio i metodi (operazioni) disponibili, i formati esatti dei messaggi XML richiesti in input e restituiti in output, e il "binding", ovvero le specifiche di rete e l'URL esatto (endpoint) su cui il servizio è in ascolto per ricevere la chiamata SOAP.
