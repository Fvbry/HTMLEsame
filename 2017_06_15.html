Domanda 1: XML

  • 1. È un linguaggio di programmazione. FALSO
  • 2. È stato progettato per descrivere dati. VERO
  • 3. Applica delle regole sintattiche semplici e rigide. VERO
Motivazione: La 1 è falsa perché XML (eXtensible Markup Language) è un meta-linguaggio di marcatura e non di programmazione (non ha cicli, variabili o logica). Le affermazioni 2 e 3 sono vere: il suo scopo è strutturare e descrivere dati utilizzando regole rigidissime (es. ogni tag aperto deve essere chiuso, ci deve essere un solo elemento radice, ecc.).

Domanda 2: JavaScript

  • 1. È un linguaggio fortemente tipato. FALSO
  • 2. Permette a una stessa variabile di avere due valori consecutivi di tipo diverso. VERO
  • 3. Supporta variabili con due scope: locale e globale. VERO
  • 4. Permette di definire funzioni richiamabili dal codice HTML. VERO
Motivazione: La 1 è falsa perché JavaScript è un linguaggio "loosely typed" (a tipizzazione debole e dinamica). Proprio per questo motivo la 2 è vera: una variabile dichiarata con `var x = 5` (numero) può diventare subito dopo `x = "Ciao"` (stringa). La 3 è vera (le variabili nelle funzioni hanno scope locale). La 4 è vera: le funzioni JS sono spesso chiamate tramite eventi HTML come onclick="".

Domanda 3: Si consideri un'applicazione REST di ricette di cucina che fornisce funzionalità per l'aggiunta di una ricetta con il seguente codice

HTTP / REST
HTTP_OPERATION URL
{
  "Titolo": "Pizza",
  "Ingredienti": [
    { "Nome": "Farina", "Quantita": "600g" },
    { "Nome": "Pomodoro", "Quantita": "300g" },
    { "Nome": "Mozzarella", "Quantita": "200g" }
  ],
  "Ricetta": [ ... ]
}

N.B. Dopo l'operazione di aggiunta ogni ricetta deve avere un id univoco.
Quale delle seguenti affermazioni è vera?

  • 1. HTTP_OPERATION=POST e URL=/ricetta/piattimici/id FALSO
  • 2. HTTP_OPERATION=PUT e URL=/ricetta/piattimici/ FALSO
  • 3. HTTP_OPERATION=POST e URL=/ricetta/piattimici/ VERO
Motivazione (max 3 righe): Per aggiungere una nuova risorsa subordinata a una collezione (in questo caso le ricette), delegando al server il compito di creare e assegnare un nuovo ID univoco, si deve utilizzare il metodo POST indirizzato all'URI base della collezione (/ricetta/piattimici/).

Domanda 4: Si consideri il seguente codice CSS

CSS
<style>
p {
  margin: 100px 50px;
  padding: 25px 50px;
  border: 25px solid black;
}
</style>
  • 1. Lo stile viene applicato solo alla prima occorrenza di un elemento p. FALSO
  • 2. Personalizza il CSS box model, concetto che si applica a tutti gli elementi HTML. VERO
  • 3. Definisce lo stile di due dei quattro lati del paragrafo. FALSO
  • 4. Padding rappresenta lo spazio libero attorno al contenuto nell'elemento p. VERO
Motivazione: La 1 è falsa perché il selettore di tag p si applica in automatico a tutti i paragrafi del documento. La 3 è falsa perché la sintassi shorthand a due valori (es. margin: 100px 50px) definisce lo stile per tutti e quattro i lati (il primo valore per Top/Bottom, il secondo per Right/Left). La 4 è vera: il padding è proprio lo spazio trasparente racchiuso tra il content e il border.

Domanda a Risposta Aperta 1: Struttura HTML e Block vs Inline

Dopo aver descritto le caratteristiche principali e a cosa serve HTML, mostrando la struttura di una pagina HTML, si discuta in dettaglio la differenza tra elementi block e inline mostrando un esempio.

Soluzione:
L'HTML (HyperText Markup Language) è il linguaggio di marcatura standard utilizzato per creare e strutturare i contenuti delle pagine Web (titoli, paragrafi, link, immagini). Non è un linguaggio di programmazione, ma utilizza tag racchiusi tra parentesi angolari per definire la semantica del testo.

Struttura base: 
<!DOCTYPE html>
<html>
  <head>
    <title>Titolo Pagina</title>
  </head>
  <body>
    <!-- Contenuto visibile -->
  </body>
</html>

Elementi Block vs Inline:
- Gli elementi Block-level (es. <div>, <p>, <h1>) forzano un "a capo" prima e dopo la loro visualizzazione. Occupano sempre il 100% della larghezza disponibile del loro contenitore padre, indipendentemente dalla lunghezza del contenuto.
- Gli elementi Inline (es. <span>, <a>, <b>) non forzano l'andata a capo. Scorrono all'interno del testo occupando esclusivamente la larghezza necessaria al loro contenuto. Inoltre, non permettono l'impostazione di un'altezza o una larghezza fissa tramite CSS.

Domanda a Risposta Aperta 2: Protocollo WSDL

Si descriva il protocollo WSDL alla base dei SOAP Web Service.

Soluzione:
Il WSDL (Web Services Description Language) è un documento scritto in formato XML che svolge la funzione di "contratto" pubblico per un Web Service basato su architettura SOAP. Il suo scopo principale è comunicare al mondo esterno (ai client) cosa fa il servizio e come richiamarlo.

Un documento WSDL è strutturato in diverse sezioni fondamentali:
1. Types: Definisce i tipi di dato XML complessi utilizzati nei messaggi (spesso appoggiandosi a XML Schema / XSD).
2. Message: Definisce astrattamente la struttura dei messaggi di input e di output.
3. PortType / Interface: Raggruppa un set di operazioni (metodi) supportate dal servizio, specificando per ognuna quali "Message" entrano e quali escono.
4. Binding: Collega le operazioni astratte a un protocollo di comunicazione fisico e a un formato dati specifico (quasi sempre SOAP over HTTP).
5. Service: Specifica l'indirizzo finale (Endpoint o URL) presso il quale il Web Service è fisicamente raggiungibile e pronto a ricevere le richieste.
