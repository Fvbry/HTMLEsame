Domanda 1: Si consideri il seguente codice JavaScript

HTML / JS
<FORM NAME="modulo" onSubmit="print(); return false;">
  <SELECT NAME="menu" id="menuid">
    <OPTION ID="op1" VALUE="Paris">Paris</OPTION>
    <OPTION ID="op2" VALUE="rome">Rome</OPTION>
  </SELECT>
  <INPUT TYPE="Submit" VALUE="OK">
</FORM>

<SCRIPT language="JavaScript">
  var city = prompt("Are you European or American?", "European");
  if(city == "American") {
    window.document.modulo.menu.options[0].text="New York";
    window.document.modulo.menu.options[1].text="San Francisco";
    
    var select = document.getElementById('menuid');
    var opt = document.createElement('option');
    opt.id = "op3";
    opt.value = "Cleveland";
    opt.innerHTML = "Cleveland";
    select.appendChild(opt);
  }
</SCRIPT>

La pagina HTML mostra un menù a tendina con:

  • 1. Paris e Rome sempre presenti tra le opzioni
  • 2. New York in posizione 0 ove presente [cite: 1516-1544]
  • 3. Cleveland e Paris mai insieme [cite: 1516-1544]
  • 4. Cleveland sempre dopo Paris

Motivare le risposte (max 6 righe): (Se l'utente sceglie "American", "Paris" diventa "New York" all'indice 0, quindi New York è in pos. 0, e siccome Paris viene sovrascritto, non sono mai presenti assieme a Cleveland)

Domanda 2: Il seguente brano di codice

JAVASCRIPT
xmlhttp=new XMLHttpRequest();
xmlhttp.open("GET", url, true);
xmlhttp.send();
xmlDoc=xmlhttp.responseXML;
  • 1. Permette di scambiare dati con il server in background [cite: 1546-1554]
  • 2. Invia una richiesta HTTP GET al server [cite: 1546-1554]
  • 3. È supportato da tutti i browser moderni [cite: 1546-1554]
  • 4. Funzionerebbe correttamente con una url che punta a un file testuale diverso da XML

Domanda 3: La funzionalità drag&drop

  • 1. Rende un elemento trascinabile tramite l'attributo "draggable=true" [cite: 1555-1558]
  • 2. Permette di trascinare solo elementi testuali
  • 3. Era disponibile prima di HTML5

Domanda 4: Si consideri il seguente brano di codice

HTTP / REST
POST /hotel_room
201 CREATED
Location: /hotel_room/{id}
  • 1. È parte di una comunicazione verso un servizio REST [cite: 1559-1566]
  • 2. Comporta la creazione di una nuova hotel_room con id specificato dal richiedente
  • 3. Fornisce lo stesso risultato di una chiamata "PUT/hotel_room/{id}"
