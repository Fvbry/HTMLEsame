Domanda 3: XHTML

  • 1. È più stringente di HTML.
  • 2. Viene introdotto per ridurre il numero di pagine web con codice HTML errato.
  • 3. Combina HTML con XML.
  • 4. Ha regole sintattiche meno rigide di HTML.

Domanda 4: XML-RPC

  • 1. Sono gli antenati di SOAP e REST per lo sviluppo di applicazioni distribuite.
  • 2. Non forniscono funzionalità per una comunicazione sicura.
  • 3. Supportano i soli tipi complessi Array e Struct.
  • 4. Utilizzano un protocollo non standard per il trasporto.

Domanda 5: In JavaScript

  • 1. È possibile definire variabili con due scope: locale e globale.
  • 2. Non è possibile definire procedure.
  • 3. Le funzioni vengono chiamate fornendo la lista dei parametri attuali.
  • 4. L'accesso e la modifica di oggetti è permesso tramite proprietà e funzioni degli oggetti.
