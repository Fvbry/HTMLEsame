Quale affermazione è corretta riguardo il codice della form?

  • 1. Propone una form con 3 campi di testo FALSO
  • 2. Il testo "enter email" è scritto in verde FALSO
  • 3. Non è definito il colore della label submit del bottone FALSO
  • 4. Non è corretto per cui l'output non è visualizzabile FALSO
Motivazione:
1. I campi sono di tipo text, email e password, non tutti di tipo testo.
2. La pseudo-classe input::placeholder definisce il colore blue, non verde.
3. Il colore del testo nel bottone è esplicitamente definito dalla regola color: #ffffff; (bianco).
4. Il codice HTML e CSS è corretto e verrà visualizzato senza errori.

  • 1. Ogni richiesta HTTP è eseguita in isolamento VERO

Domanda 4: Si consideri il seguente script JavaScript, quale sarà l'output?

JAVASCRIPT
var b=1;
function outer() {
  var b=2;
  function inner() {
    b++;
    var b=3;
    console.log(b);
  }
  inner();
}
outer();
  • 1. 1 FALSO
  • 2. 2 FALSO
  • 3. 3 VERO
  • 4. 6 FALSO
Motivazione (max 3 righe): A causa del meccanismo di hoisting, la dichiarazione var b viene sollevata all'inizio della funzione inner() come undefined. L'istruzione b++ opera su undefined dando NaN, dopodiché viene assegnato b=3; pertanto il log stamperà 3.

Domanda a risposta aperta

Dato il seguente codice HTML/Javascript descrivere sinteticamente il significato delle righe numerate. Si dia anche una descrizione generica della pagina.

HTML / JS
<html>
<head>
<title>Di tutto un po'...</title>
<script type="text/JavaScript">
function check() {
  if (navigator.appName != "Microsoft Internet Explorer")
    alert("Attenzione, la pagina potrebbe non essere visualizzata correttamente");
}
function pieno () {
  dato_in = document.f.x.value;
  if (dato_in) {
    for (var i=0; i < dato_in.length; i++) {
      if (dato_in.charAt(i) != "")
        return true;
    }
  }
  return false;
}
</script>
</head>
<body onLoad="check()">
  <script type="text/JavaScript">
    oggi = new Date();
    document.write("<h2>" + oggi.getDate() + "/" + (oggi.getMonth() + 1) + "/" + oggi.getYear() + "</h2>");
  </script>
  <br />
  <form name="f" method="get" action="/cgi-bin/elabora.tcl">
    <input type="text" name="x" size="10"><br />
    <input type="submit" value=" Invia">
  </form>
  <img src="verifica.jpg" width="50" height="50" onmouseover="if (! pieno()) alert('Errore!')" />
  <br /><br /><a href="JavaScript: history.go(-1)"><img width="50" height="25" src="b.gif" /></a>
</body>
</html>
Soluzione:
Generico: La pagina è un modulo interattivo con controlli lato client in JavaScript per la validazione di una form e la stampa della data.

- function check(): All'evento onLoad del tag body, verifica l'identità del browser. Se non è Internet Explorer, mostra un popup di avviso.
- document.write(...): Stampa dinamicamente all'interno della pagina la data odierna formattandola in formato GG/MM/ANNO all'interno di un'intestazione h2.
- <form name="f" ... >: Modulo che invia in GET il campo testo "x" a uno script lato server ("elabora.tcl").
- function pieno() / onmouseover: L'immagine verifica.jpg possiede un trigger che si attiva al passaggio del mouse. Invoca la funzione pieno() che itera sul contenuto dell'input testuale per verificare se l'utente ha inserito almeno un carattere; se è vuoto solleva un alert di Errore.
- history.go(-1): L'ultima immagine funge da link che sfrutta le API del browser per far tornare indietro l'utente di una pagina nella cronologia di navigazione.
