Domanda 1: Quale affermazione è corretta riguardo il codice della form?

CSS / HTML
form { width: 200px; padding: 60px 100px; }
p { color: red; font-size: 16px; font-weight: 600; }
input::placeholder {color: blue;}
button { background: green; color: #ffffff; padding: 15px 64px; }
...
<form>
  <input type="text" placeholder="Enter Name"/>
  <input type="email" placeholder="Enter email"/>
  <input type="password" placeholder="Enter Password"/>
  <button>Submit</button>
</form>
  • 1. Propone una form con 3 campi di testo FALSO (Sono text, email, password)
  • 2. Il testo "enter email" è scritto in verde FALSO (I placeholder sono blu)
  • 3. Non è definito il colore della label submit del bottone FALSO (È definito color: #ffffff)
  • 4. Non è corretto per cui l'output non è visualizzabile FALSO (Il codice è sintatticamente valido)

Nota: Le opzioni sono tecnicamente tutte false in base a una lettura rigorosa del codice fornito.

Domanda 3: Il concetto di statelessness nei servizi RESTful significa che

  • 1. Ogni richiesta HTTP è eseguita in isolamento VERO
  • 2. Il server usa informazioni da richieste precedenti, nella stessa sessione FALSO
  • 3. Il client inserisce tutto ciò che è necessario a soddisfare la richiesta VERO
  • 4. I servizi web utilizzano HTTP per la gestione della sessione FALSO (Non c'è sessione lato server)

Domanda 4: Output script JavaScript

JAVASCRIPT
var b=1;
function outer() {
  var b=2;
  function inner() {
    b++;
    var b=3;
    console.log(b);
  }
  inner();
}
outer();
  • 1. 1
  • 2. 2
  • 3. 3 CORRETTA
  • 4. 6
Motivazione: A causa del meccanismo di Hoisting in JavaScript, la dichiarazione `var b` all'interno di `inner()` viene "sollevata" all'inizio della funzione. Quando `b++` viene eseguito, `b` è locale ma ancora undefined (restituendo NaN). Subito dopo viene assegnato `b=3`, quindi il `console.log(b)` stamperà sempre 3.

Domanda a Risposta Aperta: Descrivere il codice HTML/JS

Dato il codice HTML/Javascript descrivere sinteticamente il significato... e una descrizione generica della pagina.

Soluzione:
La pagina è un semplice modulo interattivo con controlli di validazione lato client.

- La funzione check() (richiamata all'onLoad del body) verifica il browser dell'utente e mostra un alert se non è Internet Explorer.
- Lo script nel body stampa dinamicamente la data odierna formattata come H2 (giorno/mese/anno).
- La pagina contiene una Form che invia tramite metodo GET il testo inserito a uno script server-side (elabora.tcl).
- C'è un'immagine che, al passaggio del mouse (onmouseover), richiama la funzione pieno(). Questa funzione controlla se l'input di testo "x" è vuoto; se è vuoto, la negazione !pieno() fa scattare un alert('Errore!').
