Domanda 1: Si consideri il seguente codice

JAVASCRIPT
<script type="text/javascript">
var codicefiscale = [
  {"id": 1, "cf": "123456"},
  {"id": 2, "cf": "234567"},
  {"id": 3, "cf": "345678"}
];
var elenco = [
  {"id": 1, "nome": "Mario", "cognome": "Rossi"},
  {"id": 2, "nome": "Carlo", "cognome": "Bianchi"},
  {"id": 3, "nome": "Alice", "cognome": "Verdi"}
];

sessionStorage.setItem("cf", JSON.stringify(codicefiscale));
sessionStorage.setItem("list", JSON.stringify(elenco));

var chiave = sessionStorage.key(0);
var tmp = sessionStorage.getItem(chiave);
var tmp_parsed = JSON.parse(tmp);
alert(tmp_parsed[1].cf);
</script>
  • 1. alert stampa 123456. FALSO
  • 2. alert stampa 234567. VERO (didatticamente)
  • 3. alert stampa 345678. FALSO
  • 4. alert ritorna un errore. FALSO
Motivazione (max 3 righe): Le specifiche del Web Storage non garantiscono un ordine rigoroso per le chiavi. Tuttavia, assumendo che il browser mantenga l'ordine di inserimento (o alfabetico), key(0) restituirà "cf". Di conseguenza, tmp_parsed sarà l'array codicefiscale. L'elemento in posizione [1] è il secondo oggetto (id: 2), il cui attributo cf è "234567". (Nel caso remoto `key(0)` fosse "list", l'alert stamperebbe semplicemente `undefined`, non un errore bloccante).

Domanda 2: Il seguente brano di codice

CSS
div {
  width: 100px;
  height: 100px;
  background: red;
  transition: width 2s, height 2s, transform 2s;
}
div:hover {
  width: 300px;
  height: 300px;
  transform: rotate(180deg);
}
  • 1. Definisce un'animazione basata su CSS3. VERO
  • 2. Al passare del mouse sopra l'elemento div, sposta l'elemento div verso l'alto. FALSO
  • 3. Al passare del mouse sopra l'elemento div, rimpicciolisce l'elemento div. FALSO
  • 4. Al passare del mouse sopra l'elemento div, ruota l'elemento div di 180 gradi. VERO
Motivazione: La 1 è vera (le transition e transform sono costrutti chiave introdotti con CSS3). La 2 è falsa in quanto manca qualsiasi regola di traslazione verticale (es. translateY o top). La 3 è falsa perché l'elemento div viene ingrandito, passando da 100px a 300px. La 4 è vera grazie alla regola transform: rotate(180deg).

Domanda 4: WSDL

  • 1. È un protocollo per lo scambio di messaggi tra computer sulla rete Internet. FALSO
  • 2. Definisce l'interfaccia di un servizio Web. VERO
  • 3. Include la specifica di come associare il WSDL con SOAP. VERO
  • 4. Specifica l'indirizzo di binding del servizio. VERO
Motivazione: La 1 è falsa perché WSDL (Web Services Description Language) non è un protocollo per lo "scambio" (per quello si usa HTTP o SOAP stesso), ma un linguaggio basato su XML per descrivere le caratteristiche del servizio. Le affermazioni 2, 3 e 4 descrivono in modo accurato i contenuti del file WSDL (interfaccia, binding a SOAP e indirizzo fisico/endpoint del servizio).

Domanda 5: JavaScript

  • 1. È un linguaggio fortemente tipato. FALSO
  • 2. Non permette di assegnare a una stessa variabile due valori consecutivi di tipo diverso. FALSO
  • 3. Supporta variabili con due scope: locale e globale. VERO
  • 4. Richiede che ogni istruzione termini con il punto e virgola ; . FALSO
Motivazione: La 1 e la 2 sono false perché JavaScript è loosely typed (debolmente tipato e dinamico); la stessa variabile può contenere un numero in una riga e una stringa nella riga successiva. La 4 è tecnicamente falsa perché JS implementa il meccanismo dell'Automatic Semicolon Insertion (ASI), grazie al quale può interpretare correttamente il codice anche senza il punto e virgola (sebbene sia considerata best practice utilizzarlo).

Domanda a Risposta Aperta 1: HTTP Operation e REST

Si descrivano le HTTP operation alla base delle architetture REST.

Soluzione:
Nelle architetture REST (REpresentational State Transfer), lo scambio e la manipolazione delle risorse avviene sfruttando in modo nativo le operazioni messe a disposizione dal protocollo HTTP (mappando le comuni operazioni CRUD sui database). Le principali operazioni sono:

- GET: Metodo sicuro e idempotente. Viene usato esclusivamente per richiedere/leggere lo stato e la rappresentazione di una specifica risorsa, senza comportare alcuna modifica o effetto collaterale lato server.
- POST: Metodo non idempotente. Utilizzato generalmente per creare una nuova risorsa subordinata all'interno di una collezione (es. inviando dati a un URI radice come `/utenti`). È il server a prendersi in carico la creazione e a restituire l'ID generato.
- PUT: Metodo idempotente. Viene usato per creare una nuova risorsa qualora l'ID/URI sia già conosciuto e stabilito dal client, oppure per aggiornare/sostituire per intero una risorsa preesistente. Essendo idempotente, invocare il metodo più volte produce il medesimo stato finale.
- DELETE: Metodo idempotente. Serve a rimuovere dal server la risorsa identificata dall'URI richiesto.

Domanda a Risposta Aperta 2: Ruolo dell'HTML DOM

Nell'ambito del linguaggio di scripting JavaScript, si descriva il ruolo dell'HTML DOM per la modifica degli elementi di una pagina web e si proponga un esempio del suo utilizzo.

Soluzione:
Il Document Object Model (DOM) HTML è un'interfaccia standard del W3C che rappresenta la pagina caricata dal browser sotto forma di una struttura logica ad albero, composta da nodi interconnessi (che incapsulano i tag HTML, i testi e gli attributi).

Il ruolo del DOM è fungere da fondamentale "ponte" tra il documento statico HTML e linguaggi di programmazione dinamici come JavaScript. Attraverso le API fornite dal DOM, JavaScript guadagna il potere di leggere, attraversare, aggiungere, eliminare o modificare in tempo reale qualsiasi elemento della pagina (compresi gli stili CSS), scatenando un aggiornamento visivo istantaneo senza dover ricaricare l'intera scheda (Dynamic HTML).

Esempio pratico:
Supponiamo di avere un paragrafo nella pagina: <p id="avviso">Attendere...</p>.
Con JavaScript possiamo intercettarlo nel DOM e mutarne sia il testo che il colore:
var elemento = document.getElementById("avviso");
elemento.innerHTML = "Operazione completata!";
elemento.style.color = "green";
