Domanda 1: Si consideri il seguente codice JavaScript

JAVASCRIPT
<script type="text/javascript">
var outside = 11;

function fz1 (inside) {
  return inside + outside;
}

function fz2() {
  outside = 19;
  return fz1(10);
}

alert(fz2());
</script>
  • 1. fz2 stampa 21. FALSO
  • 2. fz2 stampa 29. VERO
  • 3. fz2 ritorna un errore. FALSO
Motivazione (max 3 righe): La variabile outside è dichiarata globalmente con valore 11. All'interno di fz2(), l'istruzione outside = 19; (senza var o let) va a sovrascrivere il valore della variabile globale. Successivamente, fz1(10) somma il parametro locale (10) con il nuovo valore globale (19), restituendo 29.

Domanda 2: Si consideri il seguente codice HTML

HTML
<ul>
  <li>Coffee</li>
  <li>Tea</li>
  <li>Milk</li>
</ul>
  • 1. Viene visualizzato come tabella. FALSO
  • 2. È parte della definizione di un elenco puntato ordinato. FALSO (con nota)
  • 3. È parte di uno script JavaScript. FALSO
  • 4. Viene specificato nel tag head del documento HTML. FALSO
Motivazione: A rigor di logica, tutte le affermazioni sono false. Il tag <ul> sta per "Unordered List" (Elenco puntato NON ordinato). L'affermazione 2 usa l'ossimoro "elenco puntato ordinato" (gli elenchi ordinati si definiscono con <ol>, Ordered List). Nota: se il professore ha commesso un errore di battitura, l'opzione 2 era la risposta intesa, ma tecnicamente è formulata in modo errato. Le altre sono palesemente false (non è una tabella, non è JS, e va nel <body>, non nell'<head>).

Domanda 3: In un RESTful Web Service

  • 1. Vengono definite funzioni tramite interfacce uniformi. FALSO
  • 2. Le risorse vengono identificate tramite URI. VERO
  • 3. Ogni servizio può usare sia rappresentazioni JSON che XML. VERO
Motivazione: La 1 è falsa perché le architetture REST non definiscono "funzioni" (es. getUtente()), che è un approccio tipico di RPC/SOAP, bensì espongono "risorse" su cui si agisce tramite i metodi standard HTTP (l'interfaccia uniforme). La 2 e la 3 sono i fondamenti di REST: uso degli URI per le risorse e "Content Negotiation" per supportare formati multipli (JSON, XML, ecc.).

Domanda 4: XML vs JSON

  • 1. JSON è usato per descrivere dati strutturati che includono array. VERO
  • 2. JSON è più verboso di XML. FALSO
  • 3. XML è usato come base dei protocolli per i servizi web basati su SOAP. VERO
Motivazione: La 1 è vera (JSON supporta nativamente gli array [], a differenza di XML). La 2 è palesemente falsa (XML è molto più verboso a causa dei tag di chiusura continui). La 3 è vera (SOAP è un protocollo che si basa interamente su messaggi strutturati in XML: Envelope, Header, Body).

Domanda 5: CSS box model

  • 1. Racchiude tutti gli elementi HTML in un box. VERO
  • 2. Specifica unicamente margin, border, padding. FALSO
  • 3. Permette di definire stile, spessore e colore del margin. FALSO
  • 4. Impone lo stesso formato per tutti e quattro i lati della box. FALSO
Motivazione: La 1 è vera, ogni elemento è trattato come una scatola. La 2 è falsa perché omette il Content (il nucleo della box). La 3 è falsa: si possono definire spessore, colore e stile per il Border, ma il Margin è unicamente uno spazio trasparente esterno e non ha "colore" o "stile". La 4 è falsa perché si possono definire regole indipendenti per ogni lato (es. margin-top, padding-left).

Domanda a Risposta Aperta: HTML5 Drag & Drop

Nell'ambito del linguaggio HTML5, si discuta la funzionalità di drag&drop fornendo un esempio.

Soluzione:
La funzionalità di Drag and Drop (D&D) introdotta con HTML5 permette all'utente di cliccare e trascinare un elemento dello schermo verso un'altra area di destinazione in modo nativo, senza ricorrere a complesse librerie esterne.

Per implementarlo occorrono tre passaggi fondamentali:
1. Rendere l'elemento trascinabile: Si aggiunge l'attributo draggable="true" all'elemento di origine e si definisce un gestore per l'evento ondragstart che memorizzi i dati trascinati.
2. Preparare la zona di rilascio (Drop Zone): Di default, i browser impediscono il rilascio di elementi. Bisogna disabilitare questo comportamento intercettando l'evento ondragover (usando event.preventDefault()).
3. Gestire il rilascio: L'evento ondrop intercetta l'elemento rilasciato e ne gestisce l'appendimento nel DOM della nuova zona.

Esempio minimo:
<div id="zonaDrop" ondrop="drop(event)" ondragover="allowDrop(event)"></div>
<img id="miaImg" src="logo.png" draggable="true" ondragstart="drag(event)">

(Nel JS associato, allowDrop farà un preventDefault(), drag userà dataTransfer.setData() per salvare l'ID dell'immagine, e drop userà dataTransfer.getData() e appendChild() per spostare l'immagine nel div).
