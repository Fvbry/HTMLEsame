Domanda 1: Si consideri il seguente codice HTML/JavaScript

HTML / JS
<div id="cd">
  <div style="margin:auto 0px">
    <h3 style="margin-left: 10px">CD</h3>
    <div class="cd" id="cd_1">
      <div class="image">
        <span class="center-img"></span>
        <img height="120px" src="static/img/ACDC.png">
      </div>
      <div>
        <br>
        <h3>AC/DC Live at River Plate</h3>
        <ul>
          <li>1h 55m</li>
          <li>Hard Rock</li>
          <li>4 Dicembre 2009</li>
        </ul>
      </div>
    </div>
    <div><button onclick="hide('cd_2')">hide 1</button></div>
    <div class="cd" id="cd_2">[...]</div>
    <div><button onclick="hide('cd_1')">hide 2</button></div>
    <div class="cd" id="cd_3">[...]</div>
    <div><button onclick="hide('cd_3')">hide 3</button></div>
  </div>
</div>
<script type="text/javascript">
  function hide (div) {
    elem = document.getElementById(div);
    if (elem.style.visibility == "hidden")
      elem.style.visibility = "visible";
    else
      elem.style.visibility = "hidden";
  }
</script>

Supponendo che l'utente prema sul pulsante "Hide 1", quali delle seguenti immagini rappresenta la visualizzazione esatta? (a, b o c?)

  • Risposta corretta: L'immagine in cui il blocco "cd_2" scompare ma rimane lo spazio vuoto che occupava. CORRETTA
Motivazione: Cliccando su "hide 1", viene invocata la funzione hide('cd_2'). La funzione applica all'elemento la proprietà CSS visibility: hidden;. A differenza di display: none; (che rimuove fisicamente l'elemento dal flusso della pagina), visibility: hidden; lo rende invisibile ma mantiene intatto lo spazio geometrico che occupa nel layout.

Domanda 2: L'architettura REST

  • 1. Implementa una moderna architettura web. VERO
  • 2. Si basa su standard per la rappresentazione dei dati come XML e JSON. VERO
  • 3. Distribuisce servizi con interfacce di accesso ad hoc. FALSO
Motivazione: L'affermazione 3 è falsa perché il principio cardine dell'architettura REST (REpresentational State Transfer) è proprio l'uso di una interfaccia uniforme e standardizzata (i classici metodi HTTP: GET, POST, PUT, DELETE), e non di interfacce custom o "ad hoc" (pratica tipica invece di SOAP o RPC).

Domanda 3: Si consideri il seguente codice CSS3

CSS
<style>
div {
  width: 100px;
  height: 100px;
  background-color: red;
  position: relative;
  animation-name: example;
  animation-duration: 4s;
  animation-iteration-count: infinite;
}
@keyframes example {
  0% {background-color:red; left: 0px; top:0px;}
  50% {background-color: blue; left: 200px; top:200px;}
  100% {background-color: red; left: 0px; top: 0px;}
}
</style>
  • 1. Qualsiasi elemento div viene traslato avanti e indietro sia sull'asse delle x che delle y. VERO
  • 2. Solo il primo elemento div definito all'interno della pagina HTML cambia colore. FALSO
  • 3. L'animazione dura 4 secondi. VERO
  • 4. L'elemento div non cambia dimensione. VERO
Motivazione:
1. VERO: i valori left e top variano da 0 a 200px e tornano a 0, creando una traslazione diagonale su X e Y.
2. FALSO: Il selettore div (senza classi o ID specifici) applica lo stile a tutti gli elementi div della pagina.
3. VERO: Definito da animation-duration: 4s;.
4. VERO: Nei @keyframes non ci sono variazioni delle proprietà width e height, quindi la dimensione resta fissa a 100x100px.

Domanda 4: Le variabili JavaScript

  • 1. Sono loosely typed. VERO
  • 2. Hanno il solo scope locale. FALSO
  • 3. Hanno un tipo di dato dinamico. VERO
Motivazione: In JavaScript le variabili non richiedono la dichiarazione del tipo ("loosely typed") e possono cambiare tipo di dato in corsa ("dinamiche"). La 2 è falsa perché JavaScript supporta sia lo scope locale (all'interno di funzioni/blocchi) che quello globale.

Domanda 5: HTML

  • 1. È un linguaggio di markup. VERO
  • 2. Stabilisce il contenuto e formattazione di un sito web. FALSO
  • 3. Ha poche regole sintattiche. VERO
  • 4. Permette di implementare pagine web scambiate tramite il protocollo HTTP. VERO
Motivazione: La 2 è falsa: HTML stabilisce la struttura e il contenuto semantico, ma è il CSS a occuparsi della formattazione visiva (layout, stile, colori).

Domanda a Risposta Aperta 1: getElementById

Si descriva il funzionamento della funzione JavaScript getElementById. Si presenti inoltre un esempio dove la funzione getElementById viene richiamata a seguito di un evento onClick su un tag a scelta.

Soluzione:
La funzione document.getElementById("nomeId") è un metodo fondamentale dell'HTML DOM. Permette a JavaScript di scansionare il documento HTML alla ricerca di un elemento specifico che possiede l'attributo id indicato come argomento. Dato che gli ID devono essere univoci all'interno di una pagina, la funzione restituisce l'oggetto corrispondente (o null se non esiste), permettendo di manipolarne dinamicamente le proprietà, come il testo (innerHTML), lo stile o le classi CSS.

Esempio di utilizzo:
<p id="messaggio">Testo originale</p>
<button onclick="document.getElementById('messaggio').innerHTML = 'Testo aggiornato al click!'">Cliccami</button>

Domanda a Risposta Aperta 2: HTML5 e semantic markup

Dopo aver presentato le principali novità introdotte da HTML5, si discuta il concetto di semantic markup.

Soluzione:
HTML5 ha rivoluzionato lo sviluppo web introducendo numerose novità: il supporto nativo a file multimediali tramite i tag <audio> e <video> (senza Flash), il Canvas per il rendering di grafica 2D/3D via JS, le API per Web Storage (Local e Session Storage al posto dei vecchi cookie), e nuovi tipi di input per le form.

Il cuore di HTML5 è però il semantic markup: l'utilizzo di tag il cui nome descrive esplicitamente il significato strutturale e il ruolo del contenuto racchiuso (sia per lo sviluppatore che per il browser/motore di ricerca). Invece di usare generici <div id="header">, HTML5 offre tag semantici specifici come <header>, <nav>, <article>, <section> e <footer>. Questo approccio migliora drasticamente la leggibilità del codice, la SEO (Search Engine Optimization) e l'accessibilità per i lettori di schermo (screen reader).
