Domanda 1: Si consideri il seguente codice HTML/JavaScript

HTML / JS
<div id="movies">
  <div style="margin:auto 0px">
    <h3 style="margin-left: 10px">MOVIES</h3>
    <div class="style" id="movie_1">
      <div class="image">
        <span class="center-img"></span>
        <img height="150px" src="static/img/doctorstrange.jpg">
      </div>
      <div>
        <br>
        <h3>Doctor Strange (2016)</h3>
        <ul>
          <li>1h 55m</li>
          <li>Action, Fantasy</li>
          <li>26 Ottobre 2016</li>
        </ul>
      </div>
    </div>
    <div><button onclick="hide('movie_1')">hide 1</button></div>
    <div class="movie" id="movie_2">[...]</div>
    <div><button onclick="hide('movie_2')">hide 2</button></div>
    <div class="movie" id="movie_3">[...]</div>
    <div><button onclick="hide('movie_3')">hide 3</button></div>
  </div>
</div>

<script type="text/javascript">
function hide (div) {
  // Nota: si assume elem = document.getElementById(div);
  if (elem.style.visibility=="hidden")
    elem.style.visibility="visible";
  else
    elem.style.visibility="hidden";
}
</script>

Supponendo che l'utente prema sul pulsante "Hide 2", quali delle seguenti immagini rappresenta la visualizzazione esatta? (a, b o c?)

  • Risposta corretta: L'immagine in cui il film 2 (quello centrale) scompare, ma lo spazio vuoto da esso occupato rimane inalterato. CORRETTA
Motivazione (max 3 righe): Lo script JS manipola la proprietà CSS visibility: hidden. A differenza di display: none (che rimuove l'elemento dal flusso della pagina ricalcolando gli spazi), visibility rende l'elemento trasparente (invisibile) ma ne preserva esattamente l'ingombro fisico nel layout.

Domanda 2: Quando diverse regole CSS si applicano a uno stesso elemento

  • 1. Viene applicata una valutazione a cascata per la risoluzione dei conflitti. VERO
  • 2. Il primo ordinamento è fatto in base al peso e all'origine della specifica regola CSS. VERO
  • 3. Priorità massima viene data alle regole definite dall'autore della pagina. FALSO
  • 4. Si considera sempre la regola più specifica. FALSO
Motivazione: La 3 è falsa perché, per motivi di accessibilità, un foglio di stile con !important definito dall'utente (nel proprio browser) vince su quello dell'autore. La 4 è falsa per la parola "sempre": la specificità (calcolo di id/class/tag) interviene solo dopo aver valutato l'origine e l'importanza (peso). Se una regola meno specifica ha !important, vincerà lei.

Domanda 3: XML-RPC

  • 1. Forniscono un approccio per lo sviluppo di applicazioni distribuite. VERO
  • 2. Forniscono tutte le funzionalità per una comunicazione sicura. FALSO
  • 3. Supportano i soli tipi complessi Array e Struct. VERO
  • 4. Utilizzano il protocollo HTTP come mezzo di trasporto. VERO
Motivazione: XML-RPC non fornisce di per sé meccanismi intrinseci per una comunicazione sicura (la 2 è falsa), affidandosi a protocolli esterni (come HTTPS) o all'autenticazione a livello HTTP. Storicamente supporta i tipi scalari (int, boolean, string...) e i due tipi complessi base <array> e <struct>.

Domanda 4: HTML

  • 1. È un linguaggio di programmazione. FALSO
  • 2. Ha una sintassi più stringente di XHTML. FALSO
  • 3. Fornisce un layer semantico per la descrizione degli elementi. VERO
Motivazione: HTML è un linguaggio di markup (marcatura), non di programmazione (non ha logica, loop o variabili). XHTML è la versione XML di HTML, ed è molto più rigido e stringente (es. chiusura obbligatoria di ogni tag). La 3 è vera, specialmente in HTML5, grazie ai tag semantici come header, article, footer.

Domanda 5: Si consideri il seguente brano di codice JSON

JSON
{
  "team": [
    {
      "id": "001",
      "name": "Cleveland Cavaliers",
      "captain": "LeBron James",
      "city": "Cleveland"
    },
    {
      "id": "002",
      "name": "Golden State Warriors",
      "captain": "Stephen Curry",
      "city": "Oakland"
    }
  ]
}
  • 1. È in formato JSON. VERO
  • 2. Specifica un array di team. VERO
  • 3. Specifica un id di tipo intero. FALSO
  • 4. Specifica ogni singolo team come un array di quattro elementi di tipo stringa. FALSO
Motivazione: La 3 è falsa perché il valore dell'id è "001" (racchiuso tra virgolette), quindi è formalmente una stringa, non un intero. La 4 è falsa perché ogni singolo team è racchiuso tra parentesi graffe {}, quindi è un Oggetto JSON, non un array.

Domanda a Risposta Aperta: Operazioni REST e Idempotenza

Si descrivano le HTTP operation alla base delle architetture REST, discutendo il concetto di idempotenza.

Soluzione:
Nelle architetture RESTful, le operazioni (CRUD) sulle risorse vengono mappate sui metodi standard del protocollo HTTP. I principali sono:
- GET (Read): Recupera la risorsa senza alterarne lo stato sul server.
- POST (Create): Invia dati per creare una nuova risorsa subordinata; è il server ad assegnare l'URI.
- PUT (Update/Create): Invia dati per sostituire integralmente una risorsa esistente o crearla se il client ne definisce in anticipo l'URI.
- DELETE (Delete): Rimuove la risorsa all'URI specificato.

L'idempotenza è una proprietà matematica fondamentale nel Web: un'operazione è idempotente se eseguirla più volte consecutive produce lo stesso effetto sul server di quando la si esegue una volta sola.
- GET, PUT e DELETE sono idempotenti. Eseguire la stessa `PUT` 10 volte sovrascrive semplicemente i dati con lo stesso stato.
- POST è l'unico metodo non idempotente: se si invia 10 volte la stessa richiesta `POST /utenti`, verranno creati 10 utenti identici ma con ID diversi nel database.
