Domanda 1: Si consideri il seguente codice JavaScript [cite: 889]

JAVASCRIPT
<script language="javascript">
var f1 = "Ferrari";
function funzione1 (motogr) {
  return f1 + " " + motogr;
}
function funzione2() {
  var f1 = "Red Bull";
  return funzione1("Yamaha");
}
window.alert(funzione2());
</script>
  • 1. funzione2 stampa "Ferrari Yamaha". [cite: 902] VERO
  • 2. funzione2 stampa "Red Bull Yamaha". [cite: 903] FALSO
  • 3. funzione2 ritorna un errore. [cite: 904] FALSO
Motivazione (max 3 righe): JavaScript utilizza il lexical scoping (scope lessicale). La funzione funzione1 è dichiarata nel contesto globale, quindi fa riferimento alla variabile globale f1 ("Ferrari"). Quando funzione1 viene richiamata dall'interno di funzione2, non "vede" la variabile locale var f1 = "Red Bull", ma continua a utilizzare quella globale con cui è stata inizializzata.

Domanda 2: Il semantic markup in HTML5 [cite: 906]

  • 1. Associa un significato al contenuto. [cite: 907] VERO
  • 2. Rende una pagina web comprensibile sia all'uomo che alla macchina. [cite: 908, 909] VERO
  • 3. Migliora l'usabilità per persone con disabilità. [cite: 910] VERO
Motivazione: Tutte le affermazioni sono corrette. I tag semantici (come <header>, <main>, <nav>) arricchiscono il codice di significato (vantaggio per i crawler) e forniscono una struttura chiara agli screen reader usati da persone con disabilità visive.

Domanda 3: Si consideri il seguente codice CSS [cite: 911]

CSS
div {
  width: 100px;
  height: 100px;
  background: red;
  transition: width 2s, height 2s, transform 2s;
}
div:hover {
  width: 300px;
  height: 300px;
  transform: rotate(180deg);
}
  • 1. Definisce un'animazione basata su CSS3. [cite: 922] VERO
  • 2. Al passare del mouse sopra l'elemento div, sposta l'elemento div verso destra. [cite: 923] FALSO
  • 3. Al passare del mouse sopra l'elemento div, ingrandisce l'elemento div. [cite: 924] VERO
  • 4. Al passare del mouse sopra l'elemento div, ruota l'elemento div di 180 gradi. [cite: 925] VERO
Motivazione: La 2 è falsa perché non c'è nessuna traslazione (es. translateX o left). Il div si ingrandisce (da 100px a 300px) e ruota su sé stesso (rotate(180deg)), effetti resi fluidi dalla regola transition del CSS3.

Domanda 4: Nell'ambito delle architetture REST [cite: 926]

  • 1. Il metodo PUT permette di creare risorse subordinate. [cite: 927] FALSO
  • 2. Il metodo GET permette di ottenere lo stato di una risorsa. [cite: 928] VERO
  • 3. Con il metodo POST il server decide il nome della risorsa creata. [cite: 929] VERO
Motivazione: La 1 è falsa. È il metodo POST che viene usato per creare risorse subordinate in una collection (es. un nuovo utente dentro `/utenti/`). Il PUT agisce su un URL noto, solitamente per aggiornare/sostituire una risorsa esistente o per crearla qualora l'ID sia già stabilito dal client. La 3 è vera per lo stesso motivo: col POST il client invia i dati alla root e riceve in risposta l'ID/URI creato dal server.

Domanda a Risposta Aperta 1: CSS Box Model

Dopo aver descritto le caratteristiche principali e a cosa servono i fogli di stile CSS, mostrando la sintassi e un esempio di regola CSS, si discuta in dettaglio il CSS box model. [cite: 932]

Soluzione:
I CSS (Cascading Style Sheets) sono il linguaggio di stile usato per descrivere l'aspetto visivo e la formattazione dei documenti HTML, permettendo di separare la struttura (HTML) dalla presentazione (colori, layout, font). Una regola base si scrive come: selettore { proprietà: valore; }.
Esempio: p { color: blue; } imposta tutti i paragrafi di colore blu.

Il CSS Box Model è il fondamento geometrico per il posizionamento degli elementi. Ogni elemento HTML è renderizzato come un "box" rettangolare composto da quattro livelli concentrici (dall'interno verso l'esterno):
1. Content: L'area in cui appare il contenuto (testo/immagine). Definito da width e height.
2. Padding: Spazio interno, trasparente, che distanzia il contenuto dal bordo.
3. Border: La linea visibile che avvolge il padding.
4. Margin: Spazio esterno, trasparente, che separa il box dagli elementi HTML adiacenti.

Domanda a Risposta Aperta 2: Ruolo dell'HTML DOM

Nell'ambito della programmazione web basata su HTML e JavaScript, si discuta in dettaglio il ruolo dell'HTML DOM. Si proponga inoltre un esempio del suo utilizzo. [cite: 934, 935]

Soluzione:
Il Document Object Model (DOM) è un'API standard che modella il documento HTML caricato nel browser come una struttura ad albero, composta da nodi (che rappresentano tag, attributi e testo). Il suo ruolo è fungere da ponte tra l'HTML e i linguaggi di scripting (come JavaScript). Senza il DOM, JS non potrebbe comprendere la struttura della pagina. Tramite i metodi del DOM, JavaScript può interagire con il documento in tempo reale (Dynamic HTML): leggere, aggiungere, modificare o cancellare tag, attributi e stili CSS in base all'interazione dell'utente.

Esempio di utilizzo:
<div id="messaggio">Saluto</div>
In JavaScript possiamo modificare questo elemento chiamando l'API del DOM:
document.getElementById("messaggio").innerHTML = "Benvenuto!";
Questo cambia dinamicamente il testo a schermo senza dover ricaricare l'intera pagina web.
