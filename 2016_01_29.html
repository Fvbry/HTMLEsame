Domanda 1: Nell'ambito delle architetture REST

  • 1. Il metodo POST permette di creare risorse subordinate. VERO
  • 2. Con il metodo PUT il server decide il nome della risorsa creata. FALSO
  • 3. Il metodo GET permette di ottenere lo stato di una risorsa. VERO
Motivazione: La 1 è vera: il POST si utilizza verso la URI di una collezione (es. `/utenti`) per creare una nuova risorsa subordinata al suo interno[cite: 1623]. La 2 è falsa perché con il metodo PUT è il client a definire l'URI specifico della risorsa da creare o aggiornare (es. `PUT /utenti/5`); è invece con il POST che il server decide autonomamente l'URI/nome della nuova risorsa[cite: 1624]. La 3 è vera: GET è un metodo sicuro usato solo per la lettura dello stato[cite: 1625].

Domanda 2: Il seguente brano di codice

HTML
<tr>
  <td>Mario</td>
  <td>Rossi</td>
  <td>Via Comelico</td>
</tr>
  • 1. Viene visualizzato come un elenco puntato non ordinato. FALSO
  • 2. È parte della definizione di una tabella. VERO
  • 3. Definisce tre paragrafi di testo. FALSO
  • 4. Viene specificato nel tag head del documento HTML. FALSO
Motivazione: I tag <tr> (Table Row) e <td> (Table Data) sono utilizzati esclusivamente per definire rispettivamente una riga e le relative celle all'interno di una tabella HTML (<table>)[cite: 1627, 1628, 1633]. Il contenuto visibile va obbligatoriamente nel <body>, mai nell'<head>[cite: 1635].

Domanda 3: La direttiva @keyframes di CSS3

  • 1. Specifica colore e gradienti del testo di una pagina HTML. FALSO
  • 2. Permette di specificare delle animazioni. VERO
  • 3. È esprimibile in maniera percentuale. VERO
Motivazione: La direttiva @keyframes è il costrutto fondamentale in CSS3 per definire lo sviluppo dei fotogrammi (keyframe) di un'animazione complessa[cite: 1636, 1638]. La sintassi prevede l'uso di percentuali (es. 0%, 50%, 100%) per indicare lo stato dello stile in quel preciso istante del ciclo di animazione[cite: 1639].

Domanda 4: HTML DOM

  • 1. È uno standard che definisce come ottenere, cambiare, aggiungere o modificare elementi HTML. VERO
  • 2. È usato da JavaScript per modificare gli elementi di una pagina HTML. VERO
  • 3. Rappresenta una pagina HTML come un albero. VERO
  • 4. Ogni oggetto HTML DOM è composto solo di proprietà. FALSO
Motivazione: Le affermazioni 1, 2 e 3 descrivono perfettamente lo standard W3C del Document Object Model (DOM)[cite: 1640, 1641, 1642, 1644]. L'affermazione 4 è falsa perché gli oggetti nel DOM non possiedono solo proprietà (come innerHTML), ma espongono anche metodi (es. appendChild()) ed eventi (es. onclick) per interagire con essi[cite: 1645].

Domanda 5: Si consideri il seguente codice JavaScript

JAVASCRIPT
var globale = 5;

function fz1(locale) {
  return locale + globale;
}

function fz2() {
  var globale = 30;
  return fz1(15);
}

alert(fz2());
  • 1. fz2 stampa 45. FALSO
  • 2. fz2 stampa 20. VERO
  • 3. fz2 ritorna un errore. FALSO
Motivazione (max 3 righe): A causa dello scope lessicale di JavaScript, la funzione fz1 fa riferimento unicamente alla variabile globale presente nel suo stesso ambito di dichiarazione (quella esterna pari a 5)[cite: 1646]. Anche se fz2 definisce internamente un'altra variabile globale = 30[cite: 1651], fz1 la ignora. Il calcolo finale è quindi 15 (parametro locale) + 5 (variabile globale vera) = 20[cite: 1655].

Domanda 6: Si consideri il seguente brano JSON

JSON
{
  "idEsame": "AWC",
  "data": "29/01/2016",
  "ora": "10",
  "luogo": "Via Comelico"
}
  • 1. Il codice proposto definisce un array di quattro elementi. FALSO
  • 2. Il codice proposto definisce un insieme non ordinato di quattro elementi. VERO
  • 3. Il codice proposto definisce quattro stringhe. FALSO
Motivazione: La 1 è falsa perché la struttura racchiusa da parentesi graffe {} definisce formalmente un Oggetto JSON, non un array (che richiede le parentesi quadre [])[cite: 1659, 1665]. Secondo le specifiche ufficiali, un Oggetto JSON è proprio definito come una collezione (insieme) non ordinata di coppie chiave/valore: in questo caso ce ne sono esattamente quattro, rendendo la 2 vera[cite: 1666]. La 3 è imprecisa perché non si tratta di quattro "semplici stringhe", ma di quattro attributi strutturati in una gerarchia oggetto/proprietà[cite: 1667].

Domanda a Risposta Aperta: Fogli di stile CSS e Box Model

Dopo aver descritto le caratteristiche principali e a cosa servono i fogli di stile CSS, mostrando la sintassi e un esempio di regola CSS, si discuta in dettaglio il CSS box model.

Soluzione:
I fogli di stile CSS (Cascading Style Sheets) sono un linguaggio informatico utilizzato per definire e gestire la formattazione e l'aspetto visivo (design, layout, colori) dei documenti HTML[cite: 1669]. Il grande vantaggio dei CSS è la netta separazione tra il "contenuto/struttura" (HTML) e la "presentazione", garantendo una maggiore manutenibilità del codice web.

Sintassi ed Esempio:
Una regola CSS si compone di un selettore e di un blocco di dichiarazioni racchiuso tra parentesi graffe. Ogni dichiarazione è una coppia proprietà: valore;.
Esempio: p { color: blue; text-align: center; } (rende blu e centrato il testo di tutti i paragrafi).

Il CSS Box Model:
Il Box Model (Modello a scatola) è il principio geometrico alla base del layouting sul web. Il browser elabora ogni singolo elemento HTML renderizzandolo come una scatola rettangolare composta da quattro strati concentrici (dall'interno verso l'esterno):
1. Content: È il nucleo utile della scatola, in cui viene mostrato il testo o l'immagine. Le sue dimensioni si regolano tramite width e height.
2. Padding: Un cuscinetto invisibile (trasparente) collocato attorno al Content. Genera uno spazio di "respiro" all'interno dei confini dell'elemento.
3. Border: È il perimetro vero e proprio che racchiude Content e Padding. Può essere stilizzato rendendolo visibile (es. linea continua, tratteggiata, con spessore e colore).
4. Margin: L'involucro più esterno e trasparente. Serve a generare una spaziatura fisica per allontanare la scatola in questione da tutti gli altri elementi HTML limitrofi, evitando che si sovrappongano.
