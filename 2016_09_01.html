Domanda 1: Uno sviluppatore deve decidere come modellare i dati gestiti dalla sua applicazione web. Il suo obiettivo è di modellare i dati in maniera che siano facilmente interpretabili e che riducano il traffico da e verso l'applicazione web.

  • 1. Lo sviluppatore modellerà i dati in formato JSON. VERO
  • 2. Lo sviluppatore modellerà i dati in formato testuale. FALSO
  • 3. Lo sviluppatore modellerà i dati in formato XML. FALSO
Motivazione (max 3 righe): JSON (JavaScript Object Notation) è la scelta ideale: rispetto a XML è molto meno verboso (riduce il payload/traffico di rete) ed è nativamente interpretato dal browser tramite JavaScript senza richiedere complessi parser aggiuntivi. (L'opzione 2 è considerata falsa in quanto troppo generica: un formato testuale semplice non è strutturato e quindi non facilmente interpretabile).

Domanda 2: L'oggetto XMLHttpRequest

  • 1. È usato per scambiare dati in background con il server. VERO
  • 2. Permette di scambiare solo dati in formato XML. FALSO
  • 3. Permette di ricevere dati dopo che la pagina è stata caricata. VERO
Motivazione: XMLHttpRequest è il nucleo della tecnologia AJAX. Nonostante il nome contenga "XML", può scambiare qualsiasi formato di dati (XML, JSON, testo piano). Il suo scopo principale è proprio quello di inviare e ricevere dati dal server in background, aggiornando la pagina senza bisogno di ricaricarla.

Domanda 3: In Javascript, le funzioni

  • 1. Sono definite tramite la keyword function. VERO
  • 2. Definiscono sia funzioni che procedure. VERO
  • 3. Vengono chiamate fornendo la lista dei parametri attuali. VERO
Motivazione: Tutte vere. In JS si usa la parola chiave function. Possono agire da "funzioni" (se restituiscono un valore con return) o da "procedure" (se si limitano ad eseguire un blocco di codice senza restituire nulla esplicitamente). Per essere eseguite, devono essere invocate passando loro i parametri attuali.

Domanda 4: Il seguente brano di codice

HTML / JS
<div align="center">
  <form name="modulo">
    <input type="text" name="over" value="Where is the mouse?" size="50px">
  </form>
  
  <a href="#" 
     onMouseOver="window.document.modulo.over.value='Milan';" 
     onmouseout="window.document.modulo.over.value='Where is the mouse?'">scritta1</a>
  <BR><BR>
  
  <a href="#" 
     onMouseOver="window.document.modulo.over.value='Crema';" 
     onmouseout="window.document.modulo.over.value='Where is the mouse?'">scritta2</a>
</div>
  • 1. Intercetta il passaggio del mouse sopra ai link contenuti nella pagina. VERO
  • 2. Non mostra mai la scritta 'Crema'. FALSO
  • 3. Mostra la scritta 'Where is the mouse? Milan'. FALSO
  • 4. Assegna uno di questi valori all'oggetto input: 'Where is the mouse?', 'Milan', oppure 'Crema'. VERO
Motivazione: La 2 è falsa perché passando sul secondo link viene visualizzato "Crema". La 3 è falsa perché il codice JS sostituisce interamente il value dell'input, non concatena le stringhe (quindi si vedrà solo "Milan", non "Where is the mouse? Milan"). La 4 è vera: in base all'interazione dell'utente, la casella mostrerà sempre e solo uno di quei tre valori.

Domanda 5: Il seguente brano di codice

HTML5
<div id="div1" ondrop="drop(event)" ondragover="allowDrop(event)"></div>
<br>
<img id="drag1" src="img.jpg" draggable="true" ondragstart="drag(event)" width="336" height="69">
  • 1. Permette di trascinare un elemento img in un'altra posizione. VERO
  • 2. Utilizza l'attributo draggable="true" per rendere l'elemento img non trascinabile. FALSO
  • 3. Utilizza l'attributo ondrop="drop(event)" per definire dove l'elemento può essere rilasciato. VERO
  • 4. Vieta di trascinare l'elemento img all'interno dell'elemento div. FALSO
Motivazione: Si tratta dell'implementazione nativa HTML5 del Drag & Drop. La 2 è falsa perché draggable="true" serve proprio a *rendere* trascinabile l'elemento. La 4 è falsa perché gli attributi ondrop e ondragover posti sul tag <div> servono esattamente ad autorizzare e gestire il rilascio dell'immagine al suo interno.

Domanda a Risposta Aperta 1: Protocollo SOAP

Si descriva il protocollo SOAP alla base dei SOAP Web Service.

Soluzione:
SOAP (Simple Object Access Protocol) è un protocollo di comunicazione formale, standardizzato e basato su XML, ideato per permettere lo scambio di messaggi strutturati e tipizzati tra macchine e applicazioni eterogenee su una rete (es. da Java a .NET e viceversa). Opera in architetture distribuite tipiche dei Web Service (SOA).

Un messaggio SOAP è rigorosamente strutturato in diverse sezioni XML:
1. Envelope (Busta): È l'elemento radice obbligatorio che identifica il documento XML come un messaggio SOAP e ne delimita l'inizio e la fine.
2. Header (Intestazione): Opzionale, viene utilizzato per scambiare metadati a livello di infrastruttura, come token per l'autenticazione/sicurezza o informazioni per il routing.
3. Body (Corpo): Obbligatorio, contiene il payload (carico utile) effettivo della chiamata, ovvero i dati in input per l'operazione richiesta o i dati in output di una risposta (inclusi i messaggi Fault in caso di errore).

Generalmente SOAP viaggia incapsulato all'interno di messaggi HTTP (spesso tramite richieste POST).

Domanda a Risposta Aperta 2: CSS (Class vs ID)

Nell'ambito del linguaggio CSS, si descrivano in dettaglio le differenze tra i selettori class ed id e si proponga un esempio del loro utilizzo.

Soluzione:
Sia class che id sono selettori CSS utilizzati per applicare stili specifici ad elementi HTML personalizzati, ma differiscono radicalmente per il loro ambito di utilizzo e per la "specificità" (il peso) nella valutazione a cascata.

- ID (Identificatore Unico): Nel CSS si indica con il prefisso #. L'attributo id deve essere univoco all'interno di un'intera pagina HTML: non possono esistere due elementi con lo stesso ID. Viene usato per stilizzare elementi unici e strutturali (es. un menu principale o un footer) e ha una "specificità" (peso) superiore a quella delle classi nella risoluzione dei conflitti.
- Class (Classe): Nel CSS si indica con il prefisso . (punto). L'attributo class è riutilizzabile: la stessa classe può essere applicata a un numero illimitato di elementi diversi nella stessa pagina. È l'ideale per stili ricorrenti (es. pulsanti, testo evidenziato, box di avviso).

Esempio:
CSS:
#header_principale { background-color: blue; } (Stile per l'unico header)
.testo_importante { color: red; font-weight: bold; } (Stile da riutilizzare)
HTML:
<div id="header_principale">Benvenuti</div>
<p class="testo_importante">Attenzione all'esame!</p>
<span class="testo_importante">Non dimenticare il badge.</span>
