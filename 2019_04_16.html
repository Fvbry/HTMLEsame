Domanda 1: Uno sviluppatore deve decidere come modellare i dati gestiti dalla sua applicazione web. Il suo obiettivo è di modellare i dati in maniera che siano facilmente interpretabili e che riducano il traffico da e verso l'applicazione web.

  • 1. Lo sviluppatore modellerà i dati in formato JSON. VERO
  • 2. Lo sviluppatore modellerà i dati in formato testuale. FALSO
  • 3. Lo sviluppatore modellerà i dati in formato XML. FALSO
Motivazione: JSON (JavaScript Object Notation) è il formato ottimale perché è molto più leggero e meno verboso rispetto a XML (riducendo così il traffico di rete) ed è nativamente e facilmente interpretabile dal browser tramite JavaScript senza pesanti parser aggiuntivi.

Domanda 2: L'oggetto XMLHttpRequest

  • 1. È usato per scambiare dati in background con il server. VERO
  • 2. Permette di scambiare solo dati in formato XML. FALSO
  • 3. Permette di ricevere dati dopo che la pagina è stata caricata. VERO
  • 4. È definito lato client. VERO
Motivazione: L'opzione 2 è falsa perché, nonostante il nome contenga "XML", l'oggetto XMLHttpRequest può ricevere e trasmettere dati in diversi formati, inclusi testo piano e JSON (oggi lo standard più usato), interrogando la proprietà responseText.

Domanda 3: Si consideri il seguente codice

HTML / JS
<div align="center">
  <form name="modulo">
    <input type="text" name="over" value="Dove si trova il puntatore?" size="50px">
  </form>
  
  <a href="" onMouseOver="window.document.modulo.over.value='Destra';" 
     onmouseout="window.document.modulo.over.value='Dove si trova il puntatore?'">scritta1</a>
  <br><br>
  
  <a href="#" onMouseOver="window.document.modulo.over.value='Sinistra';" 
     onmouseout="window.document.modulo.over.value='Dove si trova il puntatore?'">scritta2</a>
</div>
  • 1. Intercetta il passaggio del mouse sopra ai link contenuti nella pagina. VERO
  • 2. Non mostra mai la scritta 'Sinistra'. FALSO
  • 3. Mostra la scritta 'Destra' insieme alla scritta 'Sinistra' quando il mouse passa velocemente da un link all'altro. FALSO
  • 4. Assegna unicamente uno di questi valori all'oggetto input: 'Destra' oppure 'Sinistra'. FALSO
Motivazione: La 2 è falsa (mostra 'Sinistra' passando su scritta2). La 3 è falsa perché l'assegnazione del valore sostituisce completamente la stringa precedente (non c'è concatenazione). La 4 è falsa perché viene assegnato anche il valore originario di default 'Dove si trova il puntatore?' quando il mouse esce dal link (evento onmouseout).

Domanda 4: Si consideri il seguente codice JavaScript

JAVASCRIPT
<script type="text/javascript">
var listino = [{"id": 1, "prezzo": "500euro"}, { "id": 2, "prezzo": "400euro"}, {"id": 3, "prezzo": "600euro"}];
var prodotti = [{"id": 1, "modello": "Galaxy S10", "marca": "Samsung"}, { "id": 2, "modello": "P20 pro", "marca": "Huawei"}, {"id": 3, "modello": "IphoneX", "marca": "Apple"}];

sessionStorage.setItem("listino", JSON.stringify(listino));
sessionStorage.setItem("prodotti", JSON.stringify(prodotti));

var chiave = sessionStorage.key(1);
var oggetto = JSON.parse(sessionStorage.getItem(chiave));
alert(oggetto[1].modello);
</script>
  • 1. alert stampa 'P20 pro'. FALSO
  • 2. sessionStorage.key(1) ritorna {'id': 2, "modello": "P20 pro", "marca": "Huawei"}. FALSO
  • 3. JSON.parse trasforma una stringa in un oggetto JavaScript. VERO
Motivazione: La 1 è falsa (potenzialmente incerta) perché l'API Web Storage non garantisce l'ordine delle chiavi salvate; key(1) potrebbe puntare a "listino" e non a "prodotti", generando undefined per .modello.
La 2 è falsa perché il metodo sessionStorage.key(indice) restituisce il *nome della chiave* (es. la stringa "prodotti" o "listino"), non il valore dell'oggetto in sé.

Domanda 5: Un RESTful service

  • 1. È stateless. VERO
  • 2. Può usare rappresentazioni multiple delle risorse. VERO
  • 3. Utilizza l'interfaccia uniforme di HTTP. VERO
Motivazione: Questi sono i vincoli fondamentali dell'architettura REST: la mancanza di sessione salvata lato server (Statelessness), la possibilità per una risorsa di essere richiesta in JSON, XML o HTML (Content Negotiation), e l'impiego dei metodi standard HTTP (GET, POST, PUT, DELETE).

Domanda 6: Le animazioni CSS3

  • 1. Permettono di modificare la posizione di un elemento. VERO
  • 2. Non permettono di definire la durata dell'animazione. FALSO
  • 3. Permettono di definire un'animazione che viene ripetuta nel tempo. VERO
  • 4. Suddividono l'animazione lungo l'asse temporale. VERO
Motivazione: La 2 è falsa perché la durata è un parametro fondamentale e si imposta esplicitamente tramite la proprietà animation-duration (es. 4s).

Domanda a Risposta Aperta: Il ruolo dell'HTML DOM

Nell'ambito della programmazione web basata su HTML e JavaScript, si discuta in dettaglio il ruolo dell'HTML DOM. Si proponga inoltre un esempio del suo utilizzo.

Soluzione:
L'HTML DOM (Document Object Model) è uno standard del W3C che modella la struttura di una pagina web sotto forma di un albero di oggetti (nodi). Il suo ruolo fondamentale è fungere da ponte (interfaccia) tra il documento HTML statico e i linguaggi di scripting come JavaScript. Tramite il DOM, JavaScript ha il potere di accedere dinamicamente alla pagina, leggere i contenuti, modificare il testo o lo stile CSS, intercettare gli eventi dell'utente, creare nuovi elementi o eliminare quelli esistenti senza dover ricaricare la pagina.

Esempio di utilizzo:
Supponiamo di avere un titolo nella pagina HTML: <h1 id="titolo">Ciao</h1>.
In JavaScript possiamo modificare il testo e il colore accedendo a quell'elemento tramite il DOM:
var elemento = document.getElementById("titolo");
elemento.innerHTML = "Benvenuto utente!";
elemento.style.color = "blue";
