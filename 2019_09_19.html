Domanda 1: Si consideri il seguente brano di codice

HTML / JS
<!DOCTYPE html>
<html>
<body>
<h1>September Exam</h1>
<div>
  <b>Student: </b> <span id="std"></span><br>
  <b>Day: </b> <span id="d"></span><br>
  <b>Hour:</b> <span id="h"></span>
  <b>Project: </b> <span id="p"></span>
</div>
<script>
  var xmlhttp, xmlDoc;
  xmlhttp = new XMLHttpRequest();
  xmlhttp.open("GET", "URL", false);
  xmlhttp.send();
  xmlDoc = xmlhttp.responseXML;
  document.getElementById("std").innerHTML= xmlDoc.getElementsByTagName("std") [0].childNodes[0].nodeValue;
  document.getElementById("d").innerHTML= xmlDoc.getElementsByTagName("d") [0].childNodes[0].nodeValue;
  document.getElementById("h").innerHTML= xmlDoc.getElementsByTagName("h") [0].childNodes[0].nodeValue;
  document.getElementById("p").innerHTML= xmlDoc.getElementsByTagName("p") [0].childNodes[0].nodeValue;
</script>
</body>
</html>
  • 1. Dopo l'esecuzione dello script, i tag span sono riempiti con valori recuperati in locale. FALSO
  • 2. Il terzo parametro della funzione xmlhttp.open indica una richiesta sincrona. VERO
  • 3. xmlhttp.responseXML contiene il risultato della richiesta effettuata. VERO
  • 4. XMLHttpRequest permette di scambiare anche documenti JSON. VERO

Domanda 5: Si consideri il seguente codice CSS

CSS / HTML
<head>
<style type="text/css">
  #container { width: 1040px; margin:0 auto; }
  #side1 { width: 700px; height: 300px; float: left; background-color: ... margin: ...; }
  #side2 { width: 300px; height: 300px; float: left; background-color: ... margin:..; }
  #side3 { clear: both; width: 1000px; height: 70px; background-color: ...; margin: ...; }
</style>
</head>
<body>
<div id="container">
  <div id="side1">side1</div>
  <div id="side2">side2</div>
  <div id="side3">side3</div>
</div>
</body>

Si proponga un disegno della struttura della pagina web accanto al codice proposto.
Motivare la risposta (max 3 righe):

Soluzione e Motivazione:
La pagina presenta un div "container" centrato (margin: 0 auto) largo 1040px. Al suo interno, "side1" (700px) e "side2" (300px) sono affiancati orizzontalmente come due colonne grazie alla proprietà float: left. Sotto di essi si posiziona "side3" (1000px), che va a capo e interrompe l'affiancamento grazie alla direttiva clear: both, fungendo da footer.

Domanda a Risposta Aperta 1: Servizi REST e CRUD

Nell'ambito dei servizi REST, si discutano in dettaglio le CRUD operation.

Soluzione:
Le operazioni CRUD (Create, Read, Update, Delete) mappano le azioni fondamentali sulle risorse RESTful utilizzando i metodi standard del protocollo HTTP:

- Create (POST): Crea una nuova risorsa subordinata; è il server a determinare l'URI della nuova risorsa.
- Read (GET): Recupera lo stato o una rappresentazione di una risorsa. È un'operazione sicura (non modifica i dati) e idempotente.
- Update (PUT): Aggiorna o sostituisce integralmente una risorsa esistente. Anche questa operazione è idempotente (eseguirla N volte ha lo stesso effetto di eseguirla una volta).
- Delete (DELETE): Rimuove la risorsa specificata dall'URI (idempotente).

Domanda a Risposta Aperta 2: CSS Box Model

Dopo aver descritto le caratteristiche principali e a cosa servono i fogli di stile CSS, mostrando la sintassi e un esempio di regola CSS, si discuta in dettaglio il CSS box model.

Soluzione:
I CSS (Cascading Style Sheets) separano il contenuto strutturale HTML dalla sua formattazione (layout, colori). La sintassi di base è: selettore { proprietà: valore; } (Es: p { color: red; }).

Il CSS Box Model è il modello geometrico fondamentale per il posizionamento degli elementi. Ogni elemento è considerato una scatola rettangolare composta da quattro strati:
1. Content: L'area centrale con i dati effettivi (testo, immagini), definita da width e height.
2. Padding: Lo spazio trasparente interno, tra il contenuto e il bordo.
3. Border: La linea di demarcazione che avvolge il padding.
4. Margin: Lo spazio trasparente esterno che separa il box dagli elementi vicini.
